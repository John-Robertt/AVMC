# ADR（架构决策记录）

ADR 用于记录“当时为什么这么定”，避免未来改动时重复争论或误解。

规则：
- 每条 ADR 只解决一个决策点
- 必须包含：背景、决策、后果（trade-off）
- 一旦落地实现，若改变决策应新增 ADR（而不是改旧 ADR）
